# 帧界成片审阅台 Design Brief

唯一规范来源为 `FJ_Final_Cut_Review_SPEC_V1.3_Reviewed.md`。
本 Brief 只把 SPEC 第 34、37、40 章转成设计约束；任何参考图、历史 Prompt
或既有 Figma 画面都不能新增功能、替代验收或覆盖 SPEC。

## 视觉方向

暗色、高密度、工业化影视审阅工作台。目标是长时间审片时可读、克制、信息密度高，而不是营销型展示页面。

非规范视觉参考重点：

- 40px 顶部栏。
- 左侧主工作区 + 固定右侧意见栏。
- 宽屏播放器中竖屏视频完整居中，两侧黑边。
- 播放器上方悬浮标注工具栏。
- 播放器右侧 150px 历史版本栏。
- 时间轴意见点、紧凑播放控制。
- 右侧意见输入、意见卡片和底部 Decision Bar。
- 选中意见必须带动播放器、时间轴点、画面批注层和意见卡片形成同一焦点状态。
- 底部成片缩略图。

## 设计原则

- 极深色底、细边框、小字号、低阴影、低圆角。
- 青绿色只作为主操作和选中态，不扩散为整页单色主题。
- 状态必须同时使用文字和图形，不只依赖颜色。
- 图标按钮必须有 Tooltip，点击区不小于 28px。
- 不出现用户头像、成员、登录、已审阅内容物理删除、撤销定稿、跨版本自动追踪 UI；项目删除只作为成片审阅入口中的安全软删除危险动作出现，必须二次确认。剪辑入口的分集条目只在审核开始前显示“删除”用于重复上传去重，并且必须二次确认；开始审核、多版本、已有意见或定稿后该按钮消失。
- `/edit` 不出现审阅写操作、项目归档、项目恢复或项目删除；`/review` 不出现项目创建、项目编辑、成片创建、成片元数据编辑、V1 上传或追加版本操作。
- 顶部入口导航采用非对称呈现：`/edit` 只显示当前“剪辑入口”，避免剪辑端暴露审阅入口；`/review` 同时显示“剪辑入口”和当前“成片审阅”，保留返回剪辑入口的路径。
- 元数据编辑保持紧凑、就地可达：项目编辑触发器放在现有项目标题栏，成片编辑触发器放在对应成片条目行；表单仅在触发后以 dialog 或 inline 方式出现，不新增独立“项目资料”“成片元数据”页面 section/card，审核工作台不展示元数据编辑。
- 项目打包只属于审阅入口；剪辑入口只允许单片定稿原片下载。
- 版本对比只能表达人工对比，不能设计自动问题对应、自动时间码映射、自动修复/遗留/新增判断。
- 不默认常驻显示当前版本全部标记；未选中意见时采用全局一致策略，本项目默认按“无选中意见不显示意见标记”设计。
- 精确回放中必须显示轻量 loading，失败时提供同入口重试，不新增解释型帮助卡片。
- 历史版本意见可以展示为参考，但要明确只读原因；点击历史意见时视觉上先切回对应版本，再呈现该版本 AnnotationSet。

## 响应式

- 1366px 及以上同时显示播放器、版本栏、意见栏。
- 小于 1280px：意见栏变抽屉，版本栏可折叠。
- 不开发手机布局。
- 关键操作始终可达，无文字重叠和关键按钮截断。

## SPEC 第 40 章增量

- `03 · Components` 需要覆盖 Issue Card 选中态、回放中、回放失败、Timeline Marker 选中/悬停、上一条/下一条禁用态。
- `05 · 审阅入口 Review` 需要覆盖当前版本精确回放、历史版本点击后切换回放、同时间码多意见只高亮当前意见。
- `06 · 状态与流程` 需要补充 `ReviewPlaybackTarget -> media ready -> seeked -> frame callback -> AnnotationSet` 的顺序。
- `08 · Contracts & Handoff` 需要明确 `ReviewPlaybackTarget`、有理数帧率、坐标归一化和竞态 sequence 不是 UI 局部状态。
- 设计交付未提供上述页面和截图 QA 证据前，不得声称已完成对当前 SPEC 的设计验收。

## 2026-07-09 UI Follow-up Design Notes

- 批注工具栏不再居中悬停，桌面工作台中右边界对齐播放器右边界，保持按钮紧凑。
- 右侧意见面板必须表现为独立工作栏：清晰边框、内部滚动条、底部与剧集列表底部齐平。
- 超宽桌面采用居中工作台，而不是全屏拉满。顶栏背景可铺满浏览器，但顶栏内容、主播放器、版本栏、意见栏、追加版本表单和版本对比区必须共享同一个非固定 1440 的流式最大宽度 token。
- 底部模块命名和信息层级为“剧集列表”：主文本为“第 N 集”，副文本为标题和版本数量；不以 V1/V2 作为列表项。
- 播放器反馈 toast 居中覆盖在播放器上，短时自动消失，不占用底部布局空间。
- 版本对比左右播放框视觉高度一致；视频只允许 `contain` 适配，不允许填充拉伸。
